# Корпоративный магазин мерча

MVP корпоративного магазина мерча, где:

- администратор начисляет коины сотрудникам;
- сотрудники могут дарить коины друг другу;
- действует лимит на подарки: не более `25` коинов в календарный месяц;
- сотрудники тратят коины в каталоге мерча;
- все операции фиксируются в журнале.

## Стек

- `Next.js`
- `TypeScript`
- `Prisma`
- `PostgreSQL`

Текущая локально запускаемая рабочая модель использует серверное JSON-хранилище в [data/app-state.json](/Users/manager/Desktop/Магазин%20мерча/data/app-state.json). Это позволяет уже сейчас иметь персистентное состояние без внешней БД. Prisma-схема остаётся целевой моделью для следующего шага миграции на `PostgreSQL`.

## Основные роли

- `ADMIN`:
  - начисляет и списывает коины;
  - управляет каталогом;
  - просматривает историю транзакций и заказов.
- `EMPLOYEE`:
  - видит баланс;
  - отправляет подарки коллегам;
  - оформляет заказы в магазине.

## Бизнес-правила

### Коины

- Начисление коинов выполняет только администратор.
- Переводы между сотрудниками доступны только сотрудникам.
- За один календарный месяц сотрудник может подарить максимум `25` коинов.
- Нельзя перевести коины самому себе.
- Нельзя потратить или перевести больше доступного баланса.

### Магазин

- У товара есть стоимость в коинах и остаток на складе.
- Заказ уменьшает баланс сотрудника и резервирует товар.
- Все изменения оформляются как транзакции, чтобы можно было строить аудит.

## Деплой на Vercel

Текущая версия проекта может быть опубликована на `Vercel` как demo-окружение.

Важно:

- проект использует серверное JSON-хранилище;
- в среде `Vercel` оно автоматически переключается на `/tmp/corporate-merch-store/app-state.json`;
- это подходит для демо и проверки интерфейса, но не для надёжного production-хранилища;
- для постоянных данных следующим шагом нужно перевести проект на `PostgreSQL / Prisma`.

### Как выложить

1. Создайте репозиторий на GitHub и загрузите проект.
2. В Vercel нажмите `Add New Project`.
3. Выберите этот GitHub-репозиторий.
4. Framework Preset: `Next.js`.
5. Build Command:

```bash
npm run build
```

6. Install Command:

```bash
npm install
```

7. Output Directory оставьте пустой.
8. Переменные окружения не обязательны для demo-режима.

### Что будет работать после деплоя

- авторизация тестовыми логинами;
- каталог;
- заказы;
- админка;
- начисления;
- реакции и история.

### Ограничение demo-режима

Если инстанс Vercel будет перезапущен, данные из JSON-хранилища могут сброситься. Для постоянной работы нужно вынести состояние в базу данных.
