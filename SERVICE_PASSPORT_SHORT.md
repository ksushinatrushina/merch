## 3. Краткая архитектура

**Описание архитектуры:**
`Монолитное web-приложение на Next.js. Состоит из frontend-интерфейса и встроенного server-side слоя (API routes) внутри одного приложения. Хранение состояния в текущем прототипе реализовано через JSON state store на файловой системе. Отдельная БД, очереди и worker-процессы пока не используются. Есть роли пользователей: сотрудник, администратор, менеджер доставки заказов.`

**Основной стек:**
`TypeScript, React 19, Next.js 15, Node.js, CSS`

**Схема взаимодействия / ссылка на диаграмму:**
`Пользователь открывает web-интерфейс → frontend вызывает встроенные API routes Next.js → серверный слой обрабатывает операции с каталогом, заказами, мерчиками, ролями и возвращает обновлённый snapshot состояния.`
